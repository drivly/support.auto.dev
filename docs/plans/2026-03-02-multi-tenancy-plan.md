# Multi-Tenancy Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make GraphDL multi-tenant by adding domain scoping to the framework and exposing it as a service via apis/.

**Architecture:** Two workstreams: (1) graphdl-orm gets a `domain` relationship field on 5 top-level collections with access control and hook forwarding, (2) apis/ gets a `/graphdl/*` route group that proxies to graphdl-orm with tenant context. Application-level tenancy readings (Customer has Domain, etc.) are already committed in auto.dev-graphdl.

**Tech Stack:** Payload CMS 3.x, Vitest, Cloudflare Workers (itty-router), MongoDB

### Task 7: Add high-level readings API

**Files:**
- Create: `C:/Users/lippe/Repos/apis/graphdl/readings.ts`
- Modify: `C:/Users/lippe/Repos/apis/index.ts`

The high-level readings endpoint accepts `{ text, multiplicity }` and orchestrates the full createFact flow (create graph-schema, create reading, set roleRelationship) via REST calls to graphdl-orm.

```typescript
export async function createReading(req: ApiRequest, env: Env) {
  const { slug } = req.params
  const { text, multiplicity } = await req.json()

  // Resolve domain ID from slug + user
  const domainId = await resolveDomain(req, env, slug)

  // Create graph schema
  const name = text.split(' ').map(w => w[0].toUpperCase() + w.slice(1)).join('')
  const schema = await post(env, 'graph-schemas', { name, domain: domainId })

  // Create reading (triggers role auto-creation)
  await post(env, 'readings', { text, graphSchema: schema.id, domain: domainId })

  // Set multiplicity
  const multMap = { '*:1': 'many-to-one', '1:*': 'one-to-many', '*:*': 'many-to-many', '1:1': 'one-to-one' }
  await patch(env, 'graph-schemas', schema.id, { roleRelationship: multMap[multiplicity] })

  return { ok: true, graphSchema: schema.id, text, multiplicity }
}
```

Register: `.post('/graphdl/domains/:slug/readings', assertAuthenticated, createReading)`

### Task 9: Add raw Payload proxy

**Files:**
- Create: `C:/Users/lippe/Repos/apis/graphdl/raw-proxy.ts`
- Modify: `C:/Users/lippe/Repos/apis/index.ts`

A catch-all proxy that forwards requests to graphdl-orm's Payload REST API, injecting the domain filter.

```typescript
export async function rawProxy(req: ApiRequest, env: Env) {
  const { collection, id } = req.params
  const domainId = req.headers.get('x-graphdl-domain')

  const url = new URL(`${env.GRAPHDL_URL}/api/${collection}${id ? '/' + id : ''}`)

  // Forward query params, inject domain filter for reads
  if (req.method === 'GET' && domainId) {
    url.searchParams.set('where[domain][equals]', domainId)
  }

  const res = await fetch(url.toString(), {
    method: req.method,
    headers: { 'Content-Type': 'application/json' },
    body: req.method !== 'GET' ? JSON.stringify({
      ...(await req.json().catch(() => ({}))),
      ...(domainId ? { domain: domainId } : {}),
    }) : undefined,
  })

  return res.json()
}
```

Register:
```typescript

.post('/graphdl/raw/:collection', assertAuthenticated, rawProxy)
.patch('/graphdl/raw/:collection/:id', assertAuthenticated, rawProxy)
```
